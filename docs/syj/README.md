# docs/syj · 사용자 ↔ AI 에이전트 소통 창구

이 폴더는 **사용자(syj)와 AI 에이전트가 서로 주고받는 공간**이다.

- **사용자 → 에이전트:** 에이전트에게 참고시킬 자료·초안·요청 사항을 여기에 올린다. "이 파일 보고 해줘"의 그 파일 자리.
- **에이전트 → 사용자:** 에이전트가 만든 산출물(사람이 읽을 문서·정리·설명·초안)을 여기에 둔다.

## `docs/`의 다른 파일들과 무엇이 다른가

| | 성격 | 대상 |
|---|---|---|
| `docs/00`~`08`, `requirements/`, `reviews/`, `LEARNINGS.md`, `OPEN-QUESTIONS.md` | 프로젝트 **거버넌스·명세·지식** | 주로 **에이전트**가 규칙·계약·이력을 참조 |
| **`docs/syj/`** | 사람과 주고받는 **작업 공간** | **사용자 ↔ 에이전트** 소통 |

즉 다른 docs는 "에이전트가 지켜야 할 것/알아야 할 것"이고, 여기 `docs/syj`는 "사람이 읽고 사람과 오가는 것"이다.

## 규칙

- 에이전트는 이 폴더에 **자유롭게 쓸 수 있다.** pre-commit 훅의 문서 잠금(`docs/0X`·`requirements`·`00`)이나
  자율 프로토콜(§8)의 "쓸 수 있는 문서" 제한이 여기엔 걸리지 않는다.
- 그래도 **비밀정보는 커밋하지 않는다**(SEC-3) — 키·토큰·비밀번호는 여기에도 두지 않는다.
- 거버넌스 문서(`CLAUDE.md`, `docs/01`~`08`, `requirements` 본문)를 고치는 것은 여전히 사람 승인 사안이다.
  여기 `docs/syj`에 초안·제안을 두는 것은 자유지만, 그것을 governance로 반영하는 건 별개다.
