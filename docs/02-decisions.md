# 02 · 결정 & 경계 (구현 제약)

> 이 문서는 **구현이 반드시 지켜야 하는 제약**이다. 각 항목을 어겨야 할 것 같으면 코드 대신 질문한다.
> S = 확정 결정, E = 등급2 경계(지금 구현 안 하되 막지 말 것).

---

## A. 확정 결정 (S)

### 색인 / 데이터

- **S1 문서 유입** — 문서는 **관리자 색인 API**로만 들어온다(UI 없음, 동기 처리, 진행률·큐 없음, 실패 시 에러 응답). 색인 로직은 진입 경로와 분리된 **서비스 계층**에 둔다.
- **S7 색인 메타데이터** — 각 조각에 필수 코어 세트를 **반드시** 보존: `document_id`, `document_name`, `location`(페이지/섹션, 없으면 순번), `access_tags[]`, `indexed_at`, `embedding_model`, `embedding_dim`.
- **S8-1 LLM 라우팅** — v0는 **단일 모델 고정**(설정값). 라우팅은 LiteLLM 책임, 코어는 모델명만 전달.
- **S8-2 문서 포맷** — v0 지원: **Tika 기본 포맷(PDF·docx·txt 등) + hwp(hwplib)**. 파서는 포맷별 어댑터.
- **S8-4 임베딩 고정** — 임베딩 모델은 **설정값으로 고정**. 조각 메타에 모델명·차원 기록. 색인/검색 임베딩은 동일해야 함(불일치 시 검색이 조용히 깨짐).
- **S11 하이브리드 검색** — **BM25 + 벡터**. 조각 원문 텍스트를 검색 가능 필드로 저장. 병합 전략(RRF 등)은 교체 가능.
- **S12 청킹** — **토큰 크기 기준 기계적 청킹**(도메인 중립). "조항 단위" 같은 도메인 청킹 금지. 청킹은 교체 가능 부품.
- **S15 한국어 분석기** — ES `chunk_text` BM25 분석기 = **nori**. (언어 설정이지 도메인 특화 아님. 언어별 교체 가능.)
- **S16 원본 보관** — 업로드 원본을 **보관**(v0=로컬 FS 볼륨), `document_id`로 연결. 재색인은 원본에서 수행. 저장 백엔드 교체 가능.
- **S17 재업로드/교체** — 업로드 시 **문서 키**(doc_key) 제공. 같은 키면 통째 교체. **교체 순서: 신버전 색인 완료 → 그다음 구버전 조각 삭제**(장애 시 문서 증발 방지). 실패는 깨끗한 실패, 데이터 유실 없음.
- **S18 접근 태그 원천** — 접근 태그의 **유일한 원천은 document**. 조각의 태그는 색인 시 복사된 **사본**. 검색 필터는 사본을 보되 진실은 document.

### 인증 / 권한

- **S3 권한 모델** — **태그 기반 필터 + 역할→태그 매핑 분리**. 문서엔 `access_tags`(중립 문자열), 사용자는 역할→태그 매핑으로 태그 획득. 필터: 사용자 태그 ∩ 문서 태그 ≠ ∅. v0 역할 2개(USER/ADMIN), 태그 2개(public/restricted). **사용자 태그와 문서 태그는 단일 공유 어휘**(별도 변환 계층 없음).
- **S4 앞단 게이트** — 접근 태그 확정은 **Security 필터체인 RBAC 단계**(앞단 게이트)에서. RAG Advisor·검색은 태그를 **소비만**, 권한 판단 금지. (RAG 우회 경로가 생겨도 권한 누락 불가)
- **S25 인증(스택)** — Keycloak OIDC. 비번·MFA는 Keycloak 위임.

### 채팅 / 스트리밍 / 이력

- **S2 세션·이력** — 세션 = 사이드바 채팅 1개, 사용자 소유(생성·조회·삭제). LLM 컨텍스트는 최근 N턴(설정값). 자동 만료 없음, 사용자 삭제만. 검색 쿼리는 마지막 질문 그대로. 세션·메시지에 생성·갱신 타임스탬프 필수.
- **S6 스트리밍 프로토콜** — **이벤트 타입 SSE**: `text`/`sources`/`error`/`done`. `sources`는 **서버 검색 결과에서 직접** 생성(LLM 파싱 금지).

### 감사

- **S5 감사 로그** — 대화 이력과 **별도 저장·독립 수명주기**. 세션 FK 없음(사용자 삭제와 무관하게 유지). 레코드: 추적ID·요청자ID·타임스탬프·근거·질문·응답. 기록 범위(전문/최소)는 설정 가능.

### 인프라 / 배포

- **S9 DB 분담** — PostgreSQL: 사용자·세션·이력·감사. Elasticsearch: 조각·벡터·메타. (통합 금지)
- **S10 Rate Limiting** — 필터체인 순서(…→RateLimit) 유지·구현, v0는 실질 정책 없이 자리만.
- **S13 리액티브-블로킹 격리** — WebFlux 유지 + **블로킹 JPA 접근을 전용 리포지토리 계층에 격리**(별도 스레드풀, 예: boundedElastic). 스트리밍 완료 시 이력·감사 저장도 격리 스케줄러로 넘기고 **스트림은 대기하지 않음**. 저장 실패는 응답 성공과 독립(로깅).
- **S8-3 폴백** — v0는 **깨끗한 실패**: LLM/ES 장애 시 명확한 에러 + 로그 + `error` 이벤트로 스트림 종료. 자동 우회·페일오버 없음.
- **S14 06 레이어** — 문서 생성·편집(06)은 v0 스코프 밖(로드맵). 단 접점을 막지 않음(E13 등).
- **S26/S27 배포** — 회사별 설치형(독립 인스턴스), Docker Compose 패키징. (v0 개발 시 로컬 compose로 재현)

## B. 등급2 경계 (E) — 지금 구현하지 않되, 막지 말 것

각 경계는 **모듈 인터페이스를 이렇게 설계하라**는 지침이다.

- **E1** 색인 로직은 진입 경로(API/CLI/UI)와 분리된 서비스로. 어떤 진입도 같은 서비스 호출.
- **E2** LLM 컨텍스트 조립은 교체 가능한 독립 컴포넌트(`ContextAssembler`). 컨트롤러에 하드코딩 금지.
- **E3** 검색 쿼리 생성은 독립 단계. 향후 쿼리 재작성을 앞에 삽입 가능.
- **E4** 역할→태그 매핑은 단일 컴포넌트(`RoleTagMapper`). 교체해도 재색인 불요.
- **E5** 감사 기록 범위(전문/최소)는 설정으로 조절. 범위 변경이 스키마 변경을 요구하지 않음.
- **E6** 메타데이터 스키마는 확장 가능. 신규 필드 추가가 기존 색인을 깨지 않음.
- **E7** 모델 지정은 게이트웨이 파라미터일 뿐. 코어에 특정 모델 하드코딩 금지.
- **E8** 문서 파서는 포맷별 교체·추가 가능 어댑터(`DocumentParser`). 신규 포맷 추가가 파이프라인 구조를 안 바꿈.
- **E9** 임베딩 변경은 재색인 수반(문서화). 메타의 모델명·차원으로 재색인 대상 식별.
- **E10** 검색 결과 병합 전략(`ResultMerger`) 교체 가능. 색인 구조 불변.
- **E11** 청킹 전략(`ChunkingStrategy`) 교체 가능. 교체 시 재색인 수반.
- **E12** 블로킹 DB 접근은 전용 리포지토리 계층에 격리. R2DBC 전환·스레드풀 조정이 이 계층 안에서.
- **E13** ChatClient 호출은 RAG Advisor 체인에 비강결합. RAG 없는 순수 호출(구조화 출력 등) 가능.
- **E14** 원본 저장 백엔드(`FileStorage`) 교체 가능. 로컬 FS → 오브젝트 스토리지.
- **E15** 문서 권한 변경 시 조각 태그 갱신은 전체 재색인 아닌 태그 필드 부분 업데이트로.
- **E16** 검색 분석기(`SearchAnalyzer`)는 언어별 교체 가능. 다국어 시 언어별 인덱스 전략은 그때 결정.

## C. 결정 간 정합성 주의점 (교차 제약)

구현 시 아래 조합에서 특히 주의:

- **재색인 × 장애 (S17 × S8-3)** — 반드시 "신버전 색인 성공 후 구버전 삭제" 순서. 역순 금지.
- **스트리밍 × 블로킹 저장 (S6 × S13)** — 스트림 완료 지점의 저장은 격리 스케줄러로. 논블로킹 파이프라인에서 블로킹 저장 직접 호출 금지.
- **태그 공간 일치 (S3 × S4 × S18)** — 사용자 태그와 문서 태그는 같은 어휘. 두 태그 체계를 따로 만들지 말 것.
- **임베딩 일관성 (S8-4)** — 색인 때와 검색 때 임베딩 모델이 반드시 같아야. 다르면 에러 없이 검색 품질만 붕괴.
