# REQ-SEARCH · 하이브리드 검색 + RBAC 필터

## 책임
질문에 대해 접근 태그로 필터된 하이브리드 검색(BM25+벡터)을 수행하고, 근거(sources)를 생성한다.

## 관련 결정
S3, S4, S8-4, S11, S15, S18 · 경계 E3, E9, E10, E16 · NFR SEC-2, PERF-3

## 인터페이스
- `SearchAnalyzer` — 언어별 분석기. v0=nori. (E16)
- `ResultMerger` — BM25·벡터 순위 결합(RRF 등). 교체 가능. (E10)
- `QueryBuilder` — 질문 → 검색 쿼리. 독립 단계(향후 쿼리 재작성 삽입). (E3)
- `EmbeddingClient` — 질문 임베딩(색인과 **동일 모델**).

## 기능 명세
1. **쿼리 생성** — v0는 마지막 사용자 질문 그대로(쿼리 재작성 없음). `QueryBuilder`가 독립 단계.
2. **질문 임베딩** — 색인과 동일한 임베딩 모델로 벡터화(S8-4).
3. **하이브리드 검색** — 단일 ES 쿼리로 BM25(nori 분석 `chunk_text`) + 벡터(cosine) 결합. `ResultMerger`로 순위 병합. top-k 설정값.
4. **접근 태그 필터** — 요청 컨텍스트의 사용자 태그(AUTH가 확정) ∩ 조각 `access_tags` ≠ ∅. **검색 단계에서 필터**(응답에서 제거 아님). 권한 없는 조각은 애초에 결과에 없음.
5. **sources 생성** — 검색된 조각에서 근거 목록 생성(문서명·위치 등). **이것이 응답 근거의 원천**. LLM 출력에서 근거를 만들지 않는다(S6).

## 불변식
- 사용자 태그와 겹치지 않는 조각은 결과에 절대 포함되지 않는다.
- sources는 실제 검색된 조각에서만 나온다.
- 검색 임베딩 모델 = 색인 임베딩 모델.
- 권한 판단을 이 모듈에서 하지 않는다(태그는 AUTH가 확정한 것을 소비만).

## 테스트 시나리오 (TDD)
- [x] 한국어 질문("연차휴가")에 조사 붙은 문서("연차휴가는")가 검색됨(nori 형태소).
- [x] 정확한 용어·코드로 물으면 BM25가 해당 문서를 잡음.
- [x] 의미 유사(휴가↔연차) 질문에 벡터가 관련 문서를 잡음.
- [x] USER({public})가 restricted 조각을 검색 → 결과에 없음.
- [x] ADMIN({public,restricted})는 restricted 조각을 검색 → 결과에 있음.
- [x] sources의 각 항목이 실제 색인된 조각과 대응(지어낸 것 아님).
- [x] 검색 임베딩 모델이 설정값(색인과 동일).
- [x] `ResultMerger` 교체 시 색인 재생성 없이 병합만 바뀜.

## 하지 않을 것 (v0)
쿼리 재작성, 리랭킹(cross-encoder), 멀티 컬렉션, 다국어 인덱스. — 인터페이스 뒤에 자리만.
